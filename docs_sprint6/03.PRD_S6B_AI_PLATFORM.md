## 3. ModelRouter Service

```text
interface ModelRouter {
  route(taskCode: TaskCode, context: ContextPackage): Promise<AiTextResult>

  // Internal chain:
  // 1. Check aiGateway.enabled config
  // 2. If enabled → AI Gateway (9router custom provider)
  // 3. If 9router fails → Workers AI fallback
  // 4. If Workers AI fails → fallback model 2
  // 5. If all fail → deterministic safe template
  // 6. Log to HL_modelRuns (every attempt)
}
```

### Provider Chain

```text
Primary  : AI Gateway (9router custom provider, model: deepseek-v4-flash-free)
Fallback1: Workers AI (@cf/meta/llama-3.3-70b-instruct-fp8-fast)
Fallback2: Workers AI (@cf/meta/llama-3.1-8b-instruct)
Fallback3: Deterministic safe template (no LLM)
```

### Config

```text
aiGateway.enabled = true
aiGateway.primaryProvider = 9router
aiGateway.retry.maxRetries = 2
aiGateway.retry.timeoutMs = 10000
aiGateway.fallback.enabled = true
```

## 4. AI Gateway Call Pattern

AI Gateway **tidak** menggunakan Service Binding. Dipanggil via REST API:

```text
POST https://gateway.ai.cloudflare.com/v1/{accountId}/{gatewayId}/{provider}/chat/completions
Headers:
  Authorization: Bearer {CLOUDFLARE_API_TOKEN}
  Content-Type: application/json
Body: standard OpenAI-compatible chat completion payload
```

Required secrets (Cloudflare env, never in D1):
```text
CLOUDFLARE_ACCOUNT_ID     — for AI Gateway URL construction
CLOUDFLARE_API_TOKEN      — for AI Gateway auth

Cloudflare_Account_ID — stored in Cloudflare env vars (never in D1 or docs)
Cloudflare_token       — stored in Cloudflare Secrets (never in D1 or docs)
```

## 7. Model Run Logger

Every AI call writes to `HL_modelRuns`:

```text
Fields:
- userId, requestId, sessionId
- channel (web|whatsapp|internal)
- taskCode, providerCode, modelCode
- promptVersion, usedVectorContext, usedAiSearch
- inputTokenCount, outputTokenCount, latencyMs
- status (success|timeout|error|safety_blocked|fallback)
- fallbackUsed, safetyDecision, safetyFlagsJson
- errorCode, createdAt
```

Logging happens:
- Before call: INSERT with status='pending' (optional, for streaming)
- After call: UPDATE with status + token counts + latency
- If safety blocked: INSERT/UPDATE with status='safety_blocked'

## 8. Prompt Version Loader

```text
Flow:
1. Request arrives with taskCode.
2. Check KV (key: prompt:{taskCode}:active) → contentHash.
3. If KV miss → query D1 HL_promptVersions WHERE promptCode=? AND status='active' ORDER BY activatedAt DESC LIMIT 1.
4. Return contentText + version + contentHash.
5. Cache to KV with TTL 300s.
```

## 10. Acceptance Criteria

```text
1. ModelRouter routes to AI Gateway (9router) as primary path.
2. If 9router fails, falls back to Workers AI.
3. If Workers AI fails, falls back to deterministic safe template.
4. Every model call logged to HL_modelRuns with correct status.
5. Model run includes operatingMode field (from clinicalCopilot.operatingMode config).
6. Prompt version loaded from KV cache first, D1 fallback.
7. Prompt content varies by operating mode (standard/proactive/super_aktif per §0.3).
8. KV cache TTL honored for all key types.
9. 9router API key never logged to D1 (only referenced via secretRef).
10. AI Gateway logs minimize medical payload (truncate > 500 chars).
```
