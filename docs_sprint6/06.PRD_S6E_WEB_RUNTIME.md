## 1. Goals

Membuat AI chat web yang benar-benar jalan dari app. Proxy route di #1, orchestrator di #2, UI di web/.

## 2. Scope

## 3. API — Proxy Flow

```text
Frontend POST /api/ai/clinical/message
  → isehat-api-worker (#1)
    → Auth check (hlSession cookie)
    → Consent check (aiConsent=1)
    → Entitlement check (feature.aiClinicalCopilot.use)
    → Quota check (against plan matrix §13.2)
    → Rate limit check (30/min per user)
    → Service Binding call to isehat-ai-worker (#2):
        → Intent classifier
        → Deterministic red flag precheck
        → ClinicalContextPackageBuilder (Sprint 6D)
        → Prompt Builder (load from KV/D1, inject context)
        → ModelRouter (Sprint 6B)
        → MedicalSafetyRuntime v2 (13 detectors)
        → Response Formatter (inject disclaimer footer, build context trace)
        → Store to HL_aiClinicalMessages
        → Log to HL_modelRuns
    ← Return formatted response to frontend
```

## 4. Session Lifecycle

```text
1. User opens /ai-clinical → frontend calls POST /api/ai/clinical/session/start
2. Worker creates HL_aiClinicalSessions row (status='active')
3. User sends messages → POST /api/ai/clinical/message
   - Each message pair (user + assistant) stored in HL_aiClinicalMessages
4. User closes session → POST /api/ai/clinical/sessions/:sessionId/close
5. Or auto-expire: cron sets status='expired' after 30 min idle
```

## 6. Allowed Output Types (11 output types)

| answerType | Description |
|---|---|
| safe_summary | Ringkasan data kesehatan user |
| possible_explanations | Daftar kemungkinan penyebab (non-final) |
| follow_up_questions | Pertanyaan lanjutan yang relevan |
| missing_data | Data apa yang kurang untuk analisis lebih baik |
| first_aid_guidance | Panduan P3K dari protokol terkurasi |
| emergency_guidance | Arahan darurat dari deterministic rule |
| doctor_handoff | Ringkasan untuk dokter |
| caregiver_summary | Ringkasan untuk caregiver (dengan permission) |
| medication_adherence_summary | Ringkasan kepatuhan minum obat |
| medication_questions_for_doctor | Pertanyaan untuk ditanyakan ke dokter/apoteker |
| blocked_unsafe_request | Permintaan yang DIBLOKIR oleh Safety Runtime |

**FORBIDDEN (mode-dependent):** diagnosis_final, prescription_guidance, dosage_instruction, specialist_claim (standard: all 4 blocked; proactive: 3 blocked; super_aktif: 0 blocked). **ALWAYS FORBIDDEN (all modes):** medication_change.

## 7. UI Components

### AiClinicalChatPage

### ContextTraceDrawer
```text

```

### SafetyDisclaimerBox
```text
- Always visible below every AI response
- Contains full disclaimer text
- Cannot be dismissed
```

## 8. Disclaimer UX

```text
1. Disclaimer always appended as footer to every medical response.
2. Visible below AI response text.
3. Cannot be dismissed, minimized, or hidden.
4. Bilingual: ID + EN.
5. Always-on — no acknowledgement gate, no opt-in.
6. Mode-specific disclaimers appended if operating mode is proactive or super_aktif (§0.3).
```

## 9. Acceptance Criteria

```text
1. /api/ai/clinical/session/start creates session, returns sessionId.
2. /api/ai/clinical/message returns AI response with disclaimer + context trace.
3. Safety Runtime blocks/rewrites per operating mode (standard: diagnosis/resep/dosis/specialist blocked; proactive: diagnosis allowed; super_aktif: diagnosis/resep/dosis allowed; medication_change always blocked).
4. Entitlement/consent/quota/rate limit checked at #1 before proxy.
5. Model run logged with correct status, tokens, latency, operatingMode.
6. Failed model → deterministic safe template returned (never blank).
7. Clinical messages encrypted at rest (contentEncrypted column).
8. UI renders chat bubbles, input box, disclaimer footer, context trace drawer.
9. Disclaimer footer always visible on every response.
```
