## 1. Goals

Membuat WhatsApp AI chatbot via Baileys yang memakai orchestrator yang sama dengan web. Tambah isehat-webhooks-worker.

| # | Item | Worker | Detail |

## 3. Architecture

```text
WhatsApp User
  → WhatsApp Cloud API
    → Baileys Gateway (VPS: Docker + PM2, Node.js 22+)
      → HTTPS via Cloudflare Tunnel
        → isehat-webhooks-worker (#4)
          → Signature validation (WA_GATEWAY_SECRET)
          → Idempotency check (providerMessageId)
          → Linked user lookup (whatsappNumberHash)
          → Forward to isehat-ai-worker (#2) via Service Binding
            → Durable Object wa-session:{whatsappLinkId} (message ordering)
            → Clinical Orchestrator (shared with web)
            → Safety Runtime v2
          → Outbound response → Queue whatsapp-outbound
            → isehat-jobs-worker (#3) consumer
              → Baileys sendMessage
```

### Webhook Endpoints
```text
POST /api/whatsapp/webhook       → validate → forward to #2
POST /api/whatsapp/media/ingest  → validate → store R2 → forward to #2
POST /api/telegram/webhook       → validate → forward to #1
POST /api/billing/webhook/xendit → validate → forward to #1
GET  /api/whatsapp/health        → health check (CRON_SECRET)
```

### Webhook Signature Validation
```text
WhatsApp: validate WA_GATEWAY_SECRET header
Telegram: validate bot token + origin
Xendit: validate XENDIT_WEBHOOK_SECRET signature
```

### Idempotency
```text
- Deduplicate via HL_whatsappMessages.providerMessageId UNIQUE
- Deduplicate via HL_paymentEvents (provider, providerEventId) UNIQUE
- Duplicate → return 200 OK (no processing) to prevent retry abuse
```

## 5. Baileys Gateway (VPS)

```text
- Runtime: Node.js 22+ via PM2 + Docker Compose
- Instance: Single VPS (starter), horizontal scale when needed
- Auth: WA_GATEWAY_SECRET shared between VPS ↔ Worker #4
- Health: GET /api/whatsapp/health every 5 min
- Failover: auto-restart on crash (PM2)
- Queue: Outbound via Cloudflare Queue whatsapp-outbound
- QR pairing: display QR via admin UI or CLI for initial link
```

## 6. WhatsApp Clinical Flow

```text
1. User sends message to WhatsApp business number
2. Baileys receives → POST to #4 /api/whatsapp/webhook
3. #4 validates signature, checks idempotency
4. #4 looks up user via whatsappNumberHash:
   - Found + verified + aiEnabled → forward to #2
   - Found + not verified → respond with link instruction
   - Not found → respond with onboarding instruction (link first)
5. #2 WhatsAppSessionDO ensures sequential processing per user
6. #2 Clinical Orchestrator runs same flow as web:
   - Intent classify, Red flag precheck, Context package, Prompt, ModelRouter, SafetyRuntime
7. Response: shorter than web (max whatsappAi.maxReplyChars)
8. Disclaimer appended: "⚕️ AI bisa salah. Keputusan = tanggung jawab Anda."
9. Outbound queued to whatsapp-outbound → #3 consumer → Baileys sendMessage
```

## 7. WhatsApp UX Rules

```text
1. Short response (max 400 chars, configurable).
2. Red flag first (if detected).
3. No long paragraphs — use numbered steps.
4. Include instruction to open app for full context trace.
5. Include opt-out command: STOP AI.
6. Disclaimer appended to every medical response.
7. Emergency: abbreviated + "Hubungi 119/112/faskes terdekat."
```

## 8. STOP AI Command

```text
- User sends: STOP AI or STOP
- #2 receives command
- Sets HL_whatsappLinks.aiEnabled = 0
- Sends confirmation: "AI iSehat dinonaktifkan. Kirim START AI untuk mengaktifkan kembali."
- Subsequent messages: respond with link instruction only (like unlinked)
- User can re-enable via web app or START AI command
```

## 9. Unlinked Number Flow

```text
1. Unlinked number sends message
2. #4 finds no HL_whatsappLinks match (via whatsappNumberHash lookup)
3. HL_whatsappMessages inserted with userId=NULL, processedStatus='ignored_unlinked'
4. Response: "Selamat datang di iSehat. Untuk menghubungkan akun WhatsApp Anda, buka aplikasi iSehat > Settings > WhatsApp AI > Hubungkan."
5. No clinical data, no diagnosis, no P3K returned.
```

## 10. Acceptance Criteria

```text
1. isehat-webhooks-worker running with Service Bindings to #1, #2, #3.
2. Baileys gateway forwards WA messages to #4.
3. #4 validates signature, checks idempotency, routes to #2.
4. Linked user receives AI response via WhatsApp.
5. Unlinked user receives linking instruction only.
6. WhatsAppSessionDO preserves message order.
7. STOP AI command disables AI, START AI re-enables.
8. WA media stored in R2 after validation.
9. WhatsApp response < 400 chars with disclaimer.
10. 429 on rate limit exceeded (100/min per number).
```
