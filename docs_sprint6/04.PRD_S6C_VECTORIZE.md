## 1. Goals

Mengaktifkan AI memory personal berbasis Vectorize Free Tier. Index, query, rebuild, delete, rerank dengan namespace `user:{userId}`.

## 2. Vectorize Free Tier Limits

```text
Max vectors/index  : 10,000,000
Dimensions         : 1536 (max)
Max metadata/vector: 10 KiB
Free namespaces    : 1000 per index
Free indexes/account: 100
Embedding model    : Workers AI @cf/baai/bge-base-en-v1.5 (768-dim, free)
Per-user cap       : 500 vectors (configurable)
Alert threshold    : 80% of 10M total (8M vectors)
```

Note: Sprint 5 seed and existing Workers AI binding use `@cf/baai/bge-base-en-v1.5` (768-dim). Sprint 6 continues with this model.

## 5. VectorizeService API

```text
interface VectorizeService {
  query(userId: number, queryText: string, topK: number, minScore: number): Promise<VectorResult[]>
  insert(userId: number, sourceType: string, sourceId: string, content: string, metadata: object): Promise<string>
  delete(userId: number, vectorIds: string[]): Promise<void>
  deleteAll(userId: number): Promise<void>
  rerank(userId: number, queryText: string, documents: string[], topN: number): Promise<RerankedResult[]>
  getStatus(userId: number): Promise<MemoryStatus>
}
```

## 6. Data Indexed vs Not Indexed

### Indexed (summarized only)

```text
1. Symptom log narrative
2. Abnormal measurement session summary
3. Safety event summary
4. Doctor report summary
5. AI clinical session summary
6. Medication adherence summary
7. Hydration/cycle summary (when consent allows)
8. WhatsApp clinical chat summary
```

### NOT Indexed

```text
1. Raw secret/token
2. Cross-user data
3. Full raw prompt
4. Full raw image
5. Sensitive family data without permission
```

## 7. Per-User Vector Limit

```text
1. Max vectors per user: 500 (configurable via vectorize.maxVectorsPerUser).
2. On insert: count user's vectors in HL_vectorDocuments WHERE status='indexed'.
3. If count >= limit → evict oldest vector (LRU, set status='deleted').
4. Eviction raises HL_safetyEvents (severity='low').
5. User can rebuild to repopulate with fresh data.
```

## 8. HL_vectorDocuments Mapping

Every Vectorize vector maps 1:1 to `HL_vectorDocuments` row (existing Sprint 5C table):
```text
- userId, vectorId, sourceType, sourceId
- contentPreview (safe text, no raw data)
- status (pending|indexed|failed|deleted|skipped)
- embeddingModel, indexTimestamp
```

## 9. Acceptance Criteria

```text
1. Every vector has namespace user:{userId}.
2. Client cannot override namespace.
3. Every vector maps to HL_vectorDocuments row.
4. Delete memory deletes vectors + sets metadata status='deleted', not D1 source data.
5. Rebuild is idempotent (rerun = same result, no duplicate vectors).
6. Embedding model version recorded in HL_vectorDocuments.
7. Vectorize failure never blocks emergency guidance.
8. Context trace shows safe preview only (from HL_vectorDocuments.contentPreview).
9. Per-user limit enforced; LRU eviction works.
10. Admin alert when total index reaches 80% capacity.
```
