## 4. Deterministic Emergency Escalation Engine

### Inputs
```text
HL_metricRules (existing Sprint 1-5)
HL_measurementValues (existing)
HL_symptomLogs (existing Sprint 5A)
HL_safetyEvents (existing Sprint 5A)
HL_userProfiles (existing)
HL_firstAidProtocols (new Sprint 6, seed 10 protocols)
```

### Flow
```text
1. Run red flag precheck BEFORE any LLM call.
2. Check HL_metricRules → severity = 'emergency'?
3. Check HL_symptomLogs → redFlagTriggered = 1?
4. Check HL_safetyEvents → severity = 'emergency'?
5. If emergency = true:
   - DO NOT call free-form LLM
   - Return emergency_template_only
   - Insert HL_safetyEvents row
   - Insert HL_auditLogs row
   - WhatsApp: send abbreviated emergency message
6. If emergency = false:
   - Allow LLM to generate guidance
   - AI cannot downgrade severity determined by rules
```

### Emergency Template
```text
⚠️ PERINGATAN DARURAT
Berdasarkan data yang tercatat, Anda memiliki tanda bahaya yang memerlukan perhatian medis segera.
JANGAN menunda. Segera hubungi:
- Layanan Darurat: 119 / 112
- Fasilitas Kesehatan terdekat
- Contact caregiver Anda (jika tersedia)
```

## 6. UI Components

### EmergencyGuidanceCard
```text
- Visually dominant (red/orange background)
- No auto-dismiss
- Call emergency/faskes CTA button
- Contact caregiver CTA button (if consent)
- AI guidance appears BELOW deterministic content
- Disclaimer always visible
```

### FirstAidProtocolCard
```text
- Title + protocol category icon
- DO steps (green, numbered)
- DON'T steps (red, numbered)
- SEEK HELP NOW (orange, numbered)
- Red flags banner at top
- "Protocol reviewed by: [reviewerStatus=approved]" footer
```

## 7. Acceptance Criteria

```text
1. isehat-jobs-worker running with cron + queue bindings.
2. Emergency red flag precheck runs before every LLM call.
3. Emergency = true → emergency_template_only (no LLM freeform, no severity downgrade).
4. Emergency event logged to HL_safetyEvents + HL_auditLogs.
5. All 10 first-aid protocols seeded in ID and EN.
6. P3K response has Do/Don't/SeekHelp sections.
7. Red flag displayed at top of all P3K output.
8. Severe conditions → direct emergency guidance, not self-treatment.
9. All P3K content has contentVersion + reviewerStatus=approved.
10. WhatsApp emergency response is abbreviated (max 400 chars) + suggests emergency contact.
```
