## 5. Prompt Version Manager

### Endpoints
```
GET    /api/admin/ai/prompt-versions        → list all prompts
POST   /api/admin/ai/prompt-versions        → create new version
PUT    /api/admin/ai/prompt-versions/:id/activate → activate version
GET    /api/admin/ai/prompt-versions/:id    → read version content
```

### Prompt Codes
```text
clinical_copilot        → main clinical chat prompt
symptom_interview       → symptom interview prompt
first_aid               → first aid guidance prompt
emergency_guidance      → emergency guidance prompt
doctor_handoff          → doctor handoff prompt
caregiver_summary       → caregiver summary prompt
```

### Activation Rule
```text
1. Only one version active per promptCode.
2. Activating v2 deactivates v1 (status → deprecated).
3. KV cache invalidated on activation.
4. Audit log written: action = 'promptVersionActivated'.
```

### Reviewer Workflow
```text
1. Admin triggers eval run → queued to #3.
2. #3 runs clinical orchestrator on each case.
3. Results stored with match/mismatch scores.
4. medicalReviewer reviews mismatched cases.
5. Review: pass / fail / needs_investigation + notes.
6. Dashboard shows: pass rate, fail rate, per-category breakdown.
```

## 9. AI Operating Mode Management

### Endpoints
```
GET  /api/admin/ai/operating-mode       → read current mode
PUT  /api/admin/ai/operating-mode       → change mode (super admin only)
Auth: session + admin.aiConfig.update + super admin role
```

### Mode Values
```text
'standard'   — default, safest. No diagnosis final, no resep, no dosis, no specialist claim.
'proactive'  — AI boleh diagnosis final. No resep, no dosis, no specialist claim.
'super_aktif' — AI boleh diagnosis final, resep, dosis, specialist claim. Medication change always blocked.
```

### Mode Change Flow
```text
1. Super admin calls PUT /api/admin/ai/operating-mode { "mode": "proactive" }
2. Server validates: mode must be one of 3 allowed values.
3. If clinicalCopilot.operatingModeChangeRequiresMedicalReviewer = 'true':
   a. Create mode change request in HL_auditLogs (action='aiOperatingModeChangeRequested', status='pending_review')
   b. Notify medical reviewer queue
   c. Medical reviewer approves via POST /api/admin/ai/operating-mode/:requestId/approve
   d. Only then: update clinicalCopilot.operatingMode in HL_systemConfigs
4. If reviewer not required: directly update HL_systemConfigs.
5. Write HL_auditLogs: action='aiOperatingModeChanged', metadataJson={from: 'standard', to: 'proactive', approvedBy: reviewerUserId}
6. Invalidate KV cache for config key.
7. Mode change takes effect on next AI call (no restart needed).
```

### Mode Change Constraints
```text
1. Only super admin (role='super_admin') can request mode change.
2. Mode change must be logged to HL_auditLogs before, during, and after.
3. Medical reviewer CANNOT change mode directly — only approve/reject requests.
4. Rolling back to 'standard' from higher mode does NOT require reviewer approval (safer direction).
5. Mode change is GLOBAL — affects all users immediately.
6. Rate limit: 1 mode change per hour (prevent rapid toggling).
```
