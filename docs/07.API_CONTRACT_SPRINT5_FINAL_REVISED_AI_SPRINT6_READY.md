# 1. Contract Goals

API contract ini adalah kontrak final untuk Sprint 5 setelah sinkronisasi dengan schema final dan revisi scope AI: fitur AI dokter / AI Doctor-like Clinical Copilot tidak dibangun sebagai fitur user-facing Sprint 5, melainkan dipindah ke Sprint 6. Sprint 5C hanya menyiapkan AI Clinical Infrastructure, Vectorize foundation, config, safety guardrail, context retrieval, memory lifecycle, dan readiness contract untuk Sprint 6.

Sprint 5 terdiri dari:

| Phase | Scope API |
|---|---|
| Sprint 5 Foundation | RBAC, subscription, entitlement, quota, admin, config, billing, audit |
| Sprint 5A | Google OAuth, Daily Health Hub, Education Cards, Symptom Log, Red Flag |
| Sprint 5B | Hydration target, water logs, dashboard hydration widget |
| Sprint 5C | AI Clinical Infrastructure & Vectorize Foundation for Sprint 6 AI Clinical Copilot readiness |
| Sprint 5D | Cycle access, settings, calendar, logs, guardrails, privacy |
| Sprint 5E | Telegram hydration reminder, inline webhook, idempotent callback |

Contract ini menghindari inkonsistensi berikut:

```text
1. Sprint 5 non-metric safety event tidak dimasukkan ke HL_alerts.
2. Red flag symptom, cycle irregularity, overhydration, dan Telegram security event memakai HL_safetyEvents.
3. Education progress memakai HL_userEducationProgress sebagai tabel resmi Sprint 5.
4. Secret value tidak dikirim balik oleh API dan tidak disimpan sebagai plaintext D1 config.
5. Config secret memakai HL_configMetadata + envVarName / secretRef.
6. Semua permission yang dipakai endpoint tersedia di SQL_SEED_SPRINT5_FINAL_REVISED_AI_SPRINT6_READY.sql.
7. Audit log memakai kolom existing Sprint 1–4: userId, action, entityType, entityId, metadataJson.
8. Sprint 5C tidak membuka AI diagnosis, AI emergency authority, prescription engine, medication dosage engine, atau AI medical oracle.
9. Endpoint Sprint 5C harus menolak atau mengabaikan clinicalCopilotMode runtime sampai Sprint 6.
10. Sprint 6 AI Clinical Copilot readiness hanya boleh berupa metadata/config/status di Sprint 5, bukan klaim diagnosis atau emergency decision.
```

# 1.1 Sprint 6 AI Clinical Copilot Boundary

Sprint 6 akan menjadi phase untuk AI Doctor-like Clinical Copilot / Personal Health Intelligence. Sprint 5 hanya menyiapkan pondasinya.

Allowed in Sprint 5C:

```text
- AI config admin dan secret-safe runtime config.
- Vectorize binding dan namespace isolation.
- AI memory document metadata, indexing job, rebuild, delete, status.
- Context retrieval untuk assistant/report sebagai infrastructure validation.
- Context trace dan source transparency.
- Server-side disclaimer enforcement.
- Data sufficiency / pattern score sebagai support metadata, bukan diagnosis.
- Readiness metadata untuk Sprint 6 Clinical Copilot.
```

Deferred to Sprint 6:

```text
- AI Doctor-like Clinical Copilot UI.
- AI symptom interview / adaptive intake.
- AI differential possibility builder.
- AI risk explanation engine.
- AI missing data detector.
- AI doctor handoff generator as premium clinical experience.
- AI caregiver clinical summary.
- Clinical evaluation workflow for high-risk AI behavior.
```

Forbidden in Sprint 5:

```text
- AI final diagnosis.
- AI emergency authority.
- AI prescription or medication dosage instruction.
- AI claim to replace doctors.
- AI medical oracle claim.
- Cross-user memory retrieval.
```

# 3. Global API Rules

## 3.1 Base URL

```text
Production Worker: https://hl-health-companion.indiehomesungairaya.workers.dev
Local/Staging: according to deployment environment
```

## 3.2 Authentication

Protected endpoints use existing HTTP-only cookie session:

```http
Cookie: hlSession=<sessionToken>
```

Exceptions:

```text
GET  /api/auth/google
GET  /api/auth/google/callback
POST /api/billing/webhook/:provider
POST /api/webhook/telegram/water
POST /api/internal/cron/hydration-reminders
```

## 3.3 Authorization Guards

Every endpoint must use server-side guards. Frontend-only access control is not enough.

```text
requireAuth(c)
requirePermission(c, permissionCode)
requireEntitlement(c, featureCode)
requireQuota(c, featureCode)
requireCycleEligible(c)
requireFamilyPermission(c, ownerUserId, permissionCode)
requireInternalSecret(c, secretName)
```

## 3.4 Standard Success Envelope

```json
{
  "success": true,
  "data": {},
  "meta": {
    "requestId": "req_20260624_abc123",
    "durationMs": 123,
    "warningCode": null
  }
}
```

## 3.5 Standard Error Envelope

```json
{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Input tidak valid.",
    "details": []
  },
  "meta": {
    "requestId": "req_20260624_abc123",
    "durationMs": 55
  }
}
```

## 3.6 Pagination

```text
limit=50
cursor=<opaqueCursor>
```

Response meta:

```json
{
  "nextCursor": "cursor_next_abc",
  "hasMore": true
}
```

## 3.7 Date & Time Format

```text
Datetime: ISO-8601 string
Date: YYYY-MM-DD
Month: YYYY-MM
```

## 3.8 Audit Log Standard

All admin/security-sensitive mutation writes to existing `HL_auditLogs`.

```json
{
  "userId": 1,
  "action": "admin.aiConfig.update",
  "entityType": "HL_systemConfigs",
  "entityId": "aiTextDefaultModel",
  "metadataJson": "{\"updatedKeys\":[\"aiTextDefaultModel\"]}"
}
```

Do not use `actorId`, `targetType`, or `targetId` in implementation because existing `HL_auditLogs` uses:

```text
userId
action
entityType
entityId
metadataJson
createdAt
```

## 3.9 Sensitive Data Rule

The following data is sensitive:

```text
symptom description
red flag details
cycle logs
pregnancy status
lactation status
menopause status
AI memory context
doctor report detail
support/admin sensitive access
```

Access must require:

```text
explicit user ownership OR explicit family permission OR restricted admin permission + audit log
```

## 3.10 Secret Handling Rule

API must never return plaintext secret values.

```text
D1 HL_systemConfigs may store public/non-secret config.
D1 HL_configMetadata marks secret config metadata.
Real secret values must be Cloudflare Secrets/Env vars where possible.
API returns configured/masked/envVarName metadata, not secret value.
```

## 7.2 Google OAuth Callback

```http
GET /api/auth/google/callback
```

Auth:

```text
No hlSession required for login/register callback.
For link callback, state must contain authenticated userId.
```

Query:

```text
code=<authorizationCode>
state=<oauthState>
error=<providerErrorOptional>
```

Behavior:

```text
1. Validate state exists, not expired, not consumed.
2. Exchange code with Google token endpoint.
3. Validate ID token/userinfo.
4. Require email_verified=true.
5. Find existing HL_oauthAccounts by provider + providerSubject.
6. If found, login existing user.
7. If email conflicts with local password account and not link mode, reject auto-merge.
8. If safe, create/link HL_users + HL_oauthAccounts.
9. Create HL_sessions and set hlSession cookie.
10. Redirect to onboarding/dashboard/returnTo.
```

Response:

```http
302 Location: /dashboard
302 Location: /onboarding
302 Location: /login?error=oauth_conflict
```

Writes:

```text
HL_oauthStates
HL_oauthAccounts
HL_users
HL_sessions
HL_auditLogs action=auth.google.login entityType=HL_users entityId=:userId
```

# 9. Sprint 5C API — AI Clinical Infrastructure & Vectorize Foundation

Sprint 5C is no longer the user-facing "AI doctor" release. It is the API foundation for Sprint 6 AI Doctor-like Clinical Copilot.

Sprint 5C API goals:

```text
1. Prepare user-isolated Vectorize context.
2. Prepare AI memory lifecycle: status, rebuild, delete, admin repair.
3. Prepare context query and trace contract.
4. Prepare AI config and feature flag boundary for Sprint 6.
5. Keep existing AI Assistant and AI Report backward compatible.
6. Reject clinicalCopilotMode runtime until Sprint 6.
```

Sprint 5C API must not expose:

```text
AI final diagnosis
AI emergency decision
AI prescription
AI medication dosage instruction
AI medical oracle claim
cross-user memory retrieval
```

# 13. Security Contract

## 13.1 Admin Security

```text
/api/admin/* requires requireAuth.
/api/admin/* requires relevant permission.
Every admin mutation writes HL_auditLogs.
Secret configs are write-only/masked and should use Cloudflare Secrets/Env.
Support role cannot see sensitive medical details by default.
```

## 13.2 Subscription Security

```text
Feature gating is server-side.
Quota usage is server-side.
Billing webhook uses provider signature/secret.
Billing webhook is idempotent by provider + providerEventId.
```

## 13.3 Medical Safety

```text
Emergency severity is deterministic.
AI cannot create emergency event without deterministic trigger.
AI Clinical Copilot runtime is deferred to Sprint 6.
Sprint 5 AI endpoints cannot perform final diagnosis, emergency authority, prescription, or medication dosage instruction.
Red flag keywords are checked server-side.
Cycle guardrail is blocking and logged.
AI disclaimer is appended server-side.
```

## 13.4 Privacy

```text
Cycle/symptom/AI memory are sensitive.
Caregiver access requires HL_familyPermissions.
Admin sensitive access requires admin.sensitiveHealth.read + audit.
Vectorize namespace must be user:{userId}; client cannot override userId.
```

## 13.5 Telegram Security

```text
Water webhook must validate secret.
chatId must map to HL_telegramLinks.
callback_query.id must be idempotent.
Unknown chatId and invalid callback data must not insert water log.
```
