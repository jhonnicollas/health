# 0. Execution Purpose

This task plan is the implementation roadmap for **iSehat Content Engine CE-1**.

CE-1 must deliver a safe, auditable, Cloudflare-first internal content engine for:

```text
1. Brand Memory
2. Content Pillars
3. Campaigns
4. Idea Generator
5. Idea Approval/Rejection
6. Draft Generator
7. Draft Revision History
8. Safety Check with internal Health Classifier
9. Source References
10. Revision-bound Approval Queue
11. Markdown/Text Export
12. Audit Log
13. AI Config and Prompt Versioning
14. AI Jobs
15. AI Quota and Usage
16. Rate Limiting
17. Dashboard Summary
```

# 1. Non-Negotiable CE-1 Boundaries

The implementation agent must not build CE-2+ features.

## 1.1 Forbidden CE-1 Features

Do not implement:

```text
1. R2 asset rendering
2. Carousel PNG rendering
3. Image generation
4. Video generation
5. MP4 rendering
6. Auto-publish
7. Platform OAuth
8. External scheduler
9. API analytics import
10. Vectorize learning loop
11. Semantic duplicate detection via Vectorize
12. TikTok publishing
13. YouTube publishing
14. Facebook publishing
15. Platform webhooks
16. Public user-facing content editor
17. Multi-brand SaaS
18. Content scoring analytics
```

## 1.2 Forbidden CE-1 Routes

These routes must not exist in CE-1:

```text
/api/content/assets/*
/api/content/templates/*
/api/content/render/*
/api/content/publish/*
/api/content/schedule/*
/api/content/platform-connections/*
/api/content/webhooks/*
/api/content/analytics/import
/api/content/vectorize/*
/api/content/video/*
```

If accidentally routed, return:

```json
{
  "ok": false,
  "error": {
    "code": "NOT_FOUND",
    "message": "This feature is not part of CE-1."
  }
}
```

# 4. Sprint / Phase Mapping

CE-1 tasks are grouped into execution sprints. Each sprint has a concrete ship target and must be completed sequentially.

| Sprint | Phases | Goal | Ship Gate |
|---|---|---|---|
| CE1.1 Foundation | 0, 1, 2, 3 | Scope guard, DB schema/seeds, shared types, auth, permission, audit, integrity | Local API boots; forbidden routes absent; DB migration runs cleanly |
| CE1.2 Core Content & AI | 4, 5, 6, 7 | Core CRUD, AI infrastructure, idea workflow, draft/revision workflow | Approved idea generates draft revision 1; MockProvider jobs pass |
| CE1.3 Safety, Approval, Export & Dashboard | 8, 9, 10, 11 | Safety check, approval queue, markdown export, dashboard/AI jobs APIs | Unsafe claims blocked; approved revision exports markdown; dashboard returns data |
| CE1.4 Frontend, Security & Hardening | 12, 13, 14 | Admin UI, security hardening, regression/smoke tests | All CE-1 acceptance criteria pass; frontend smoke tests green |

Sprint rules:

```text
1. Sprints are strictly sequential: CE1.1 → CE1.2 → CE1.3 → CE1.4.
2. A phase cannot start before all previous phases in the same or earlier sprint are DONE.
3. Each task must be committed separately with task ID in the message.
4. At the end of each sprint, run the full validation commands listed in IMPLEMENTATION_GUIDE.md §7-9.
5. Update current-task.md after every task status change.
```
