## 1. Executive Verdict

PRD sebelumnya **sudah benar secara arah besar**, tetapi **belum aman untuk langsung diberikan ke agent AI sebagai dokumen eksekusi coding**.

Masalah utamanya bukan konsep. Konsepnya kuat:

- AI-generated content.
- Human-approved workflow.
- Medical safety check.
- Analytics learning loop.
- D1 sebagai source of truth.
- R2 untuk asset.
- Vectorize untuk brand memory dan learning.

Namun PRD masih memiliki banyak gap yang bisa membuat implementasi agent AI:

1. Membuat scope terlalu melebar.
2. Salah implementasi auto-publish.
3. Menyimpan token/secret dengan cara berisiko.
4. Membuat konten kesehatan yang tidak cukup aman secara compliance.
5. Gagal membangun revision history yang benar.
6. Gagal menghubungkan analytics ke draft/content secara akurat.
7. Membuat schema D1 tanpa foreign key, constraint, index, audit log, dan platform account model.
8. Menganggap “render video di Worker” terlalu mudah padahal Cloudflare Worker bukan tempat ideal untuk render MP4 berat.
9. Membuat Vectorize terlalu cepat dipakai tanpa lifecycle rebuild/delete/versioning yang jelas.
10. Tidak memiliki API contract detail, error code, status transition, dan test plan.

Kesimpulan:

```text
PRD v1.0 = Bagus sebagai concept PRD.
PRD v1.0 = Belum cukup sebagai execution PRD.
```

## 3. P0 Blockers

## P0-001 — Scope MVP Masih Terlalu Besar

### Masalah

PRD menyebut CE-1 sebagai MVP, tetapi keseluruhan dokumen tetap membawa:

- Research.
- Ideation.
- Drafting.
- Safety.
- Rendering.
- R2.
- Vectorize.
- Analytics.
- Learning.
- Scheduling.
- Publishing.
- Video.
- Multi-platform.

Ini bisa membuat agent AI mencoba membangun terlalu banyak sekaligus.

### Risiko

Agent akan membuat implementasi setengah matang:

- UI ada tapi API kosong.
- Schema ada tapi workflow tidak jalan.
- Renderer tidak stabil.
- Analytics dibuat palsu.
- Publishing API dibuat tanpa token model yang aman.

### Fix

PRD harus memaksa **MVP CE-1 hanya text engine**:

```text
CE-1 P0 Scope:
1. Brand Memory
2. Content Pillars
3. Campaign
4. Idea Generator
5. Draft Generator
6. Medical Safety Checker
7. Approval Queue
8. Manual Export Text/Markdown

CE-1 Explicitly Excludes:
1. R2 asset rendering
2. Image generation
3. Video generation
4. Auto-publish
5. Platform OAuth
6. API analytics import
7. Vectorize learning loop
```

## P0-002 — Auto-Publish Safety Rule Belum Cukup Ketat

### Masalah

PRD mengatakan health content tidak boleh auto-publish tanpa approval. Namun masih ada celah:

- Siapa yang boleh approve?
- Apakah marketingAdmin boleh approve konten kesehatan?
- Apakah safety status `warning` boleh publish?
- Apakah konten non-health boleh bypass?
- Bagaimana jika draft direvisi setelah approval?

### Risiko

Konten yang tidak aman bisa dipublish setelah revisi tanpa re-check.

### Fix

Tambahkan rule:

```text
Publishing Gate:
1. Draft must have latest safetyStatus = safe.
2. Draft revision approved must match current revision.
3. Any edit after approval resets approvalStatus to needs_review.
4. Health content requires medicalReviewer or owner approval.
5. MarketingAdmin cannot approve health safety alone.
6. Blocked content cannot be scheduled, exported as final, or published.
7. Warning content requires explicit reviewer override with reason.
```

## P0-003 — Tidak Ada State Machine untuk Draft

### Masalah

Status draft hanya `draft`, tetapi workflow sebenarnya kompleks.

### Risiko

Agent akan membuat status random dan transisi tidak konsisten.

### Fix

Definisikan state machine:

```text
idea
→ draft_generating
→ draft_ready
→ safety_checking
→ safety_safe / safety_warning / safety_blocked
→ needs_review
→ revision_requested
→ approved
→ asset_rendering
→ assetReady
→ scheduled
→ publishing
→ published
→ analytics_pending
→ analytics_collected
→ learning_processed
```

Rule:

```text
1. approved hanya boleh dari needs_review.
2. schedule hanya boleh dari approved atau assetReady.
3. edit setelah approved harus reset ke draft_ready.
4. publish hanya boleh dari scheduled.
5. safety_blocked tidak boleh lanjut.
```

## P0-004 — Revision History Disebut, Tetapi Tidak Ada Table Revision

### Masalah

PRD menyebut revision history, tapi table `conDrafts` hanya punya kolom `revision INTEGER`.

Itu bukan revision history.

### Risiko

Tidak bisa audit perubahan konten.

### Fix

Tambahkan table:

```sql
CREATE TABLE conDraftRevisions (
  id TEXT PRIMARY KEY,
  draftId TEXT NOT NULL,
  revisionNumber INTEGER NOT NULL,
  snapshotJson TEXT NOT NULL,
  changeReason TEXT,
  changedBy TEXT,
  createdAt TEXT NOT NULL,
  UNIQUE(draftId, revisionNumber)
);
```

Rule:

```text
1. Every draft edit creates new revision row.
2. Safety check is attached to exact revisionNumber.
3. Approval is attached to exact revisionNumber.
4. Asset render is attached to exact revisionNumber.
```

### Masalah

### Risiko

CREATE TABLE conPlatformConnections (

  scopesJson TEXT,
  expiresAt TEXT,
  lastRefreshAt TEXT,

  updatedAt TEXT NOT NULL

Rules:

```text

2. Token value stored in secret store / encrypted vault / environment secret binding.
3. API response may expose tokenStatus only.

5. Token refresh failure creates security/audit event.
```

## P0-009 — Content Scoring Formula Tidak Normalized

### Masalah

Formula score:

```text
contentScore =
(view_score)
+ (engagement_rate × 20)
+ ...
```

Namun `view_score`, `negative_feedback`, `followers_delta` tidak didefinisikan skala normalisasinya.

### Risiko

Score bisa lebih dari 100 atau bias besar ke akun kecil/besar.

### Fix

Definisikan normalized formula:

```text
normalized_views = min(100, views / baseline_views_for_platform * 100)
engagement_rate = engagements / max(reach, 1)
save_rate = saves / max(reach, 1)
share_rate = shares / max(reach, 1)
clickRate = clicks / max(reach, 1)

contentScore = clamp(
  normalized_views * 0.20
  + engagement_rate_score * 0.20
  + save_rate_score * 0.20
  + share_rate_score * 0.20
  + clickRateScore * 0.15
  + followerDeltaScore * 0.05
  - negative_feedback_penalty,
  0,
  100
)
```

Tambahkan baseline per platform.

## P0-010 — Research Engine Berisiko Halusinasi

### Masalah

### Risiko

AI membuat klaim kesehatan, tren, atau platform insight tanpa sumber.

### Fix

Tambahkan:

```text
Research Source Rule:
1. AI-generated research must mark sourceType = ai_inferred if no external source.
2. External research must store source_url, source_title, fetchedAt.
3. Health-related educational claims must include source_reference or be marked as general/non-clinical.
4. No statistics may be used unless source-backed.
5. All research briefs must include confidence: low / medium / high.
```

Tambahkan field:

```sql
sourceReliability TEXT CHECK(sourceReliability IN ('official','medical_reference','platform_docs','competitor','social_observation','ai_inferred'))
confidence TEXT CHECK(confidence IN ('low','medium','high'))
fetchedAt TEXT
```

## P0-011 — Rendering Video di Worker Tidak Realistis untuk MVP

### Masalah

PRD menyebut Worker/Renderer untuk MP4, tetapi video rendering dengan ffmpeg/Remotion biasanya berat dan tidak ideal di Cloudflare Workers biasa.

### Risiko

Agent mencoba menjalankan ffmpeg di Worker dan gagal.

### Fix

Pisahkan rendering:

```text
Phase 2 Renderer:
- Carousel PNG renderer can run in Worker only if template rendering is lightweight.
- For MP4, use external render worker/VPS/container.
- Cloudflare Worker only orchestrates job and stores output metadata.
- Final MP4 must be uploaded to R2 by render worker.
```

Tambahkan explicit non-goal CE-1/CE-2:

```text
No server-side MP4 rendering in Cloudflare Worker MVP.
```

# 7. Security & Privacy Leak Check

## 7.1 Secret Leakage

Status: **Belum aman penuh**

Issue:

```text
Token storage disebut, tetapi tidak ada concrete secret vault pattern.
```

Required fix:

```text
D1 may store only tokenRef and tokenStatus.
Never store platform raw token, AI key, refresh token, webhook secret, or API key in D1.
```

## 7.2 Prompt Leakage

Status: **Belum cukup**

Issue:

```text
AI prompts could accidentally include secret/config/debug data.
```

Required fix:

```text
Prompt context builder must explicitly whitelist fields.
Never pass full config rows to AI.
```

## 7.3 R2 Asset Leakage

Status: **Perlu diperkuat**

Issue:

```text
Signed URL expiry disebut, tetapi no max expiry.
```

Fix:

```text
Signed URL max expiry:
- preview: 15 minutes
- export ZIP: 60 minutes
- platform upload: single-use or shortest possible
```

## 7.4 Analytics/Comment Privacy

Status: **Belum cukup**

Issue:

```text
Audience comments can contain medical stories or PII.
```

Fix:

```text
Anonymize before storing in Vectorize.
Do not store identifiable health stories as brand memory.
```

# 13. Revised Safety Gate

## 13.1 Draft Generation Gate

```text

→ classify content type
→ if health_content=true, run medical safety checker
→ if safe, mark needs_review
→ if warning, mark needs_medical_review
→ if blocked, mark safety_blocked
```

## 13.2 Approval Gate

```text
Can approve only if:
1. currentRevision has safety report
2. safetyStatus = safe
3. reviewer has permission
4. required disclaimer exists
5. source trace exists if medical factual claims are included
```

## 13.3 Publish Gate

```text
Can publish only if:
1. draft status approved or assetReady
2. approved_revision = currentRevision
3. safety_report_revision = currentRevision
4. schedule exists
5. platform connection active
6. asset exists if platform requires media
7. no expired signed URL
```
