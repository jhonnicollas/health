# 0. Purpose

This document defines the complete test plan for **iSehat Content Engine CE-1**.

CE-1 must prove that the system can safely create, review, safety-check, approve, and export text-based social content without implementing CE-2+ features.

The test plan is written for:

```text
1. Coding agent
2. QA agent
3. Human reviewer
4. Regression test automation
5. Production smoke testing
```

---

Test only that these routes/features are absent or blocked.

# 2. Test Strategy

## 2.1 Test Levels

| Level | Purpose | Required |
|---|---|---:|
| Unit Test | Validate pure services, validators, transitions, redaction, prompt builder | Yes |
| Repository Test | Validate D1/SQLite queries, constraints, seed data | Yes |
| Integration Test | Validate service + repository + AI mock flow | Yes |
| API Contract Test | Validate endpoints, permissions, request/response, error shape | Yes |
| Security Test | Validate secret redaction, prompt whitelist, forbidden access | Yes |
| Workflow/E2E Test | Validate complete content pipeline from campaign to export | Yes |
| Smoke Test | Validate minimum production readiness after deploy | Yes |
| Regression Test | Validate known P0/P1 issues never return | Yes |

## 2.2 Test Environments

| Environment | AI Provider | DB | Publish | Expected |
|---|---|---|---|---|
| local | MockProvider | Local D1 / SQLite | Disabled | Full automated test suite |
| staging | MockProvider + optional real provider | Staging D1 | Disabled | Integration + smoke |
| production | Real or configured provider | Production D1 | Disabled in CE-1 | Smoke only after deploy |

# 3. Global Acceptance Gates

CE-1 is not accepted unless all gates below pass.

```text
[ ] Database migration runs cleanly.
[ ] Required CE-1 tables exist.
[ ] Default brand and four pillars are seeded.
[ ] No CE-2+ tables are required to run CE-1.
[ ] Every /api/content route requires authentication.
[ ] Every mutation route checks permission.
[ ] Every endpoint returns standard success/error shape.
[ ] Every AI endpoint requires Idempotency-Key.
[ ] Every AI endpoint checks rate limit before AI call.
[ ] Every AI endpoint checks quota before AI call.
[ ] PromptContextBuilder uses whitelist fields only.
[ ] No raw secret/token reaches prompt, audit log, AI job input/output, or API response.
[ ] IdeaStatus and DraftStatus are separated.
[ ] Generated ideas have non-null contentHash.
[ ] Duplicate contentHash is blocked.
[ ] Approved idea can generate draft revision #1.
[ ] Draft edit creates immutable revision and resets safety/approval.
[ ] Safety check creates exactly one safety report per draftId + revisionNumber.
[ ] Non-health content still receives safety report row.
[ ] Health classifier runs as internal sub-step of safety_check.
[ ] Only one public conAiGenerationJobs row is created for safety-check endpoint.
[ ] Source trace blocks approval when required and missing.
[ ] Health content cannot be approved by marketingAdmin alone.
[ ] medicalReviewer cannot edit draft content.
[ ] Warning approval requires override reason.
[ ] Blocked content cannot be approved or final-exported.
[ ] Final export requires approved current revision.
[ ] Final export filename includes _rev{revision}.
[ ] Audit logs are created for all sensitive mutations.
[ ] Forbidden CE-2+ routes are absent or return CE-1 NOT_FOUND.
```
