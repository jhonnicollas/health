# 0. Purpose

This document defines the API contract for **iSehat Content Engine CE-1**.

CE-1 is an internal admin application for:

```text
1. Brand Memory management.
2. Content Pillars management.
3. Campaign management.
4. Idea generation and approval.
5. Draft generation and revision history.
6. Safety check with internal health classifier.
7. Source reference management.
8. Revision-bound approval.
9. Markdown/Text export.
10. AI config, prompt version, quota, usage, job logs.
11. Dashboard and audit logs.
```

CE-1 must not implement R2 asset rendering, Vectorize learning, platform OAuth, scheduler, auto-publish, image generation, video generation, analytics import, or platform webhooks.

# 1. Global API Rules

## 1.1 Base URL

```text
/api/content
```

## 1.2 Transport

```text
Protocol: HTTPS
Request body: JSON
Response body: JSON
Datetime format: ISO-8601 UTC string
Pagination: cursor or page-based. CE-1 default uses page-based query params.
```

## 1.3 Authentication

All endpoints in this contract are protected internal admin endpoints.

Every request must be authenticated through the existing iSehat/internal auth layer.

Expected resolved user object:

```ts
type InternalUser = {
  id: string
  email?: string
  roles: Array<
    "owner" |
    "marketingAdmin" |
    "medicalReviewer" |
    "designer" |
    "aiConfigAdmin" |
    "viewer"
  >
  permissions: string[]
}
```

## 1.4 Authorization

Every endpoint must enforce server-side permission checks.

UI hiding is not security. API permission is the source of authority.

## 1.5 Standard Request Headers

```http
Content-Type: application/json
Accept: application/json
Authorization: Bearer <session-token-or-existing-auth-token>
```

If the product uses cookie/session auth, `Authorization` may be replaced by the existing session mechanism.

## 1.6 Idempotency Header

Required for AI mutation endpoints:

```http
Idempotency-Key: <unique-string>
```

Required endpoints:

```text
POST /api/content/campaigns/:id/generate-ideas
POST /api/content/ideas/:id/generate-draft
POST /api/content/drafts/:id/safety-check
POST /api/content/ai-jobs/:id/retry
```

Server behavior:

```text
1. Reject missing Idempotency-Key with VALIDATION_ERROR.
2. Store idempotency key in conAiGenerationJobs.idempotencyKey.
3. If same key and same input is repeated, return existing completed/failed job result.
4. If same key but different input is repeated, return IDEMPOTENCY_CONFLICT.
```

## 1.7 Standard Success Response

```json
{
  "ok": true,
  "data": {}
}
```

## 1.8 Standard Error Response

```json
{
  "ok": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Human readable error message",
    "details": {}
  }
}
```

## 1.9 Common Error Codes

| Code | HTTP | Meaning |
|---|---:|---|
| `UNAUTHORIZED` | 401 | Missing or invalid session |
| `FORBIDDEN` | 403 | User lacks permission |
| `VALIDATION_ERROR` | 400 | Request body/query/path is invalid |
| `NOT_FOUND` | 404 | Entity does not exist |
| `CONFLICT` | 409 | State conflict or duplicate action |
| `RATE_LIMITED` | 429 | Rate limit exceeded |
| `QUOTA_EXCEEDED` | 429 | AI quota exceeded |
| `AI_PROVIDER_FAILED` | 502 | AI provider failed or timed out |
| `SAFETY_BLOCKED` | 409 | Draft revision is safety blocked |
| `REVISION_MISMATCH` | 409 | Requested revision is not current revision |
| `APPROVAL_REQUIRED` | 409 | Approval or safety report is missing |
| `APPROVAL_PERMISSION_DENIED` | 403 | Role cannot approve this content type |
| `EXPORT_NOT_APPROVED` | 409 | Final export requested for non-approved revision |
| `SOURCE_TRACE_REQUIRED` | 409 | Safety report requires source trace but none exists |
| `PROMPT_VERSION_NOT_FOUND` | 404 | No active prompt version found |
| `AI_CONFIG_NOT_FOUND` | 404 | No active AI config found |
| `IDEMPOTENCY_CONFLICT` | 409 | Idempotency key reused with different input |

## 1.10 Pagination Query

List endpoints must support:

```text
?page=1&pageSize=20
```

Default:

```text
page=1
pageSize=20
max_page_size=100
```

Standard paginated response:

```json
{
  "ok": true,
  "data": {
    "items": [],
    "pagination": {
      "page": 1,
      "pageSize": 20,
      "total": 0,
      "hasNext": false
    }
  }
}
```

## 1.11 Sorting Query

List endpoints may support:

```text
?sort=createdAt&order=desc
```

Allowed order:

```text
asc
desc
```

## 1.12 Rate Limits

| Action | Limit |
|---|---:|
| Generate ideas | 5 batch/hour/brand |
| Generate draft | 100/day/brand |
| Safety check | 200/day/brand |
| Export markdown | 300/day/brand |
| AI config update | 20/day/brand |

Rate limit state is stored in `conRateLimitCounters`.

Rate limit error:

```json
{
  "ok": false,
  "error": {
    "code": "RATE_LIMITED",
    "message": "Rate limit exceeded.",
    "details": {
      "action": "generate_ideas",
      "resetAt": "2026-07-01T12:00:00Z"
    }
  }
}
```

## 1.13 Audit Rule

All sensitive mutations must write `conAuditLogs`.

Audit payloads must be redacted before insert.

Redacted keys:

```text
authorization
access_token
refresh_token
api_key
secret
password
token
client_secret
webhook_secret
```

```http
Idempotency-Key: required
Rate limit action: generate_ideas
Audit: idea.generate
AI job type: idea_generation
Mode: synchronous-with-job-log
```

Request:

```json
{
  "quantity": 30,
  "platforms": ["instagram", "linkedin"],
  "formats": ["carousel", "post"],
  "pillarIdsJson": ["pillar_health_data_awareness"],
  "language": "id"
}
```

Validation:

```text
1. campaign must exist and not be archived.
2. brandId must exist.
3. all pillarIdsJson must exist, be active, and belong to brand.
4. platforms must contain only instagram/linkedin.
5. formats must contain only carousel/post/story_poll/reels_script.
6. quantity must be between 1 and 50.
7. language must be id/en/bilingual.
8. Idempotency-Key required.
9. AI quota must pass before AI call.
10. Rate limit must pass before AI call.
```

AI input context:

```text
Brand Memory whitelist fields
Campaign objective/audience/language
Active selected pillars
Requested platforms/formats/quantity
```

AI output must produce:

```json
[
  {
    "title": "Tensi tinggi sekali bukan cuma soal angka",
    "pillarSlug": "health_data_awareness",

    "confidence": "medium"

]
```

```text
1. create conAiGenerationJobs row.
2. set job status running.
3. call AI provider.
4. validate JSON output.
5. generate contentHash for each idea.
6. insert conIdeas with status=idea.
7. insert conAiUsageLogs.
8. update conAiQuotas.
9. set job status completed.
10. write audit log.
```

# 20. State Transition Rules

## 20.1 Idea Transition

| Endpoint | From | To |
|---|---|---|
| `POST /ideas/:id/approve` | `idea` | `idea_approved` |
| `POST /ideas/:id/reject` | `idea` | `rejected` |

Forbidden:

```text
1. rejected -> idea_approved
2. archived -> idea_approved
3. idea_approved -> rejected unless explicitly reopened in future phase
```

## 20.2 Draft Transition

| Endpoint | From | To |
|---|---|---|
| generate draft | `draft_generating` | `draft_ready` |
| safety check start | `draft_ready` | `safety_checking` |
| safety safe | `safety_checking` | `safety_safe` -> `needs_review` |
| safety warning | `safety_checking` | `safety_warning` -> `needs_review` |
| safety blocked | `safety_checking` | `safety_blocked` |
| request revision | `needs_review` or `safety_blocked` | `revision_requested` |
| update draft | `revision_requested` or `approved` or `exported` | `draft_ready` |
| approve | `needs_review` | `approved` |
| export final | `approved` | `exported` |

Forbidden:

```text
1. safety_blocked -> approved
2. draft_ready -> approved without safety report
3. approved -> exported if approval revision != current revision
4. approved -> exported if safety report revision != current revision
5. warning -> approved without warningOverrideReason
6. sourceTraceRequired -> approved without source reference
```

# 21. Security and Privacy Contract

## 21.1 Secret Storage

D1 may store:

```text
secretRef
provider
model
purpose
non-secret config
```

D1 must not store:

```text
raw API key
raw OAuth access token
raw OAuth refresh token
webhook secret
password
platform token
client secret
```

## 21.2 API Secret Response Rule

API must never return:

```text
secretRef by default
raw secret value
Authorization header
access_token
refresh_token
api_key
client_secret
webhook_secret
password
```

Allowed:

```json
{
  "secretConfigured": true
}
```

## 21.3 Prompt Context Rule

AI prompt context must include only whitelisted fields:

```text
brand.positioning
brand.productValueJson
brand.targetAudienceJson
brand.tone
brand.forbiddenClaimsJson
brand.allowedClaimsJson
brand.disclaimerTemplate
active pillars
campaign objective
platform
format
language
```

Route handlers must not concatenate raw DB rows into prompts.

# 22. Repository Integrity Checks

Because CE-1 uses service/repository-layer referential integrity, every mutation must check:

```text
1. brandId exists before creating pillar/campaign/idea/draft/config/quota.
2. campaignId exists before creating idea/draft linked to campaign.
3. pillarId exists and active before creating idea.
4. ideaId exists and status=idea_approved before generating draft.
5. draftId exists before creating revision/safety/approval/source reference/export.
6. revisionNumber exists for draft before creating safety/approval/source reference/export.
```

If failed:

```json
{
  "ok": false,
  "error": {
    "code": "NOT_FOUND",
    "message": "Referenced entity not found.",
    "details": {
      "field": "campaignId"
    }
  }
}
```

# 23. API Acceptance Criteria

```text
[ ] Every endpoint returns standard success/error shape.
[ ] Every protected endpoint enforces auth.
[ ] Every endpoint enforces required permission.
[ ] Every mutation validates request body.
[ ] Every sensitive mutation writes audit log.
[ ] AI endpoints require Idempotency-Key.
[ ] AI endpoints check rate limit before AI call.
[ ] AI endpoints check quota before AI call.
[ ] AI endpoints create conAiGenerationJobs row.
[ ] AI endpoints log usage after attempt.
[ ] Generate ideas creates conIdeas with non-null contentHash.
[ ] Generate draft requires idea.status=idea_approved.
[ ] Generate draft creates revision #1.
[ ] Update draft creates new revision and resets safety/approval.
[ ] Safety check creates one safety report for every revision.
[ ] Non-health content gets safe safety report row.
[ ] Approval requires matching current revision.
[ ] Approval requires safety report.
[ ] Approval blocks sourceTraceRequired without source reference.
[ ] Health content cannot be approved by marketingAdmin alone.
[ ] medicalReviewer cannot edit draft content.
[ ] Warning content requires override reason.
[ ] Blocked content cannot be approved/exported.
[ ] Final export requires approved current revision.
[ ] Final export filename includes _rev{revision}.
[ ] Secret values never appear in API responses.
[ ] Prompt context never includes raw config rows or token-like fields.
```
