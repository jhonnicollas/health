# 8A. Ponytail Discipline for Stress Tooling

When writing k6, Artillery, custom Node.js, or test harness code, agents must apply the Ponytail coding skill as an implementation discipline:

```text
1. Check whether a stress script already exists before creating a new one.
2. Prefer stdlib/native test helpers and existing project dependencies.
3. Keep load scripts small, explicit, deterministic, and environment-variable driven.
4. Never remove auth/security/privacy checks just to make the load script pass.
5. Never hardcode secrets, tokens, production URLs, or real user data.
6. Do not over-engineer reusable abstractions before the scenario proves necessary.
```
