
# 9. API Contract Test Plan

## 9.1 Universal API Assertions

Every API test must assert:

```text
[ ] Response follows standard envelope: success/data/meta or success/error/meta.
[ ] Protected endpoint rejects missing/invalid hlSession.
[ ] Permission endpoint rejects insufficient permission.
[ ] Premium endpoint rejects missing entitlement when feature gated.
[ ] Quota endpoint rejects exhausted quota.
[ ] Mutation writes audit log when contract says admin/security-sensitive.
[ ] No response contains plaintext secret.
[ ] Sensitive detail is hidden unless owner/explicit permission/restricted admin + audit.
[ ] Error code matches API contract.
```
