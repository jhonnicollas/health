# 3. Global Non-Negotiable Rules

## 3.1 Medical Safety

```text
- AI cannot decide emergency.
- AI cannot diagnose definitively.
- AI cannot prescribe medication.
- AI cannot change medication dose.
- AI cannot claim to replace doctors.
- Red flag symptom detection must be deterministic server-side.
- Cycle contraception guardrail must be blocking, not toast-only.
- Overhydration is a warning, not a diagnosis.
- Medical AI output must have server-side disclaimer enforcement.
```

## 3.2 Privacy and Secret Safety

```text
- Symptom detail, red flag detail, cycle, pregnancy, lactation, menopause, AI memory, and report detail are sensitive.
- Caregiver access requires HL_familyPermissions.
- Support/admin sensitive access requires explicit permission + audit log.
- No plaintext secret in D1, seed, frontend bundle, API response, log, or audit metadata.
- Telegram webhook, billing webhook, internal cron, entitlement, quota, cycle eligibility, red flag, and disclaimer enforcement are never frontend-only.
```

## 3.3 Standard Commands

```bash
cd worker
npx tsc -p tsconfig.json
npm test


cd ../web
npx tsc -b
npx eslint .
npx vite build
```

## 3.4 Ponytail Coding Skill Rule

Ketika coding, agent harus memakai skill Ponytail dari:

```text
https://github.com/DietrichGebert/ponytail
```

Ponytail rule untuk Sprint 5:

```text
1. Jangan tulis kode baru jika fitur tidak perlu dibuat.
2. Cari implementasi existing di codebase sebelum membuat komponen/service baru.
3. Pakai standard library/native platform sebelum menambah dependency.
4. Pakai dependency existing sebelum menambah package baru.
5. Pilih perubahan minimum yang memenuhi task.
6. Jangan memotong security, medical safety, validation, accessibility, privacy, audit, idempotency, atau tests.
7. Jika solusi bisa 1 line/1 helper tanpa mengurangi safety, pilih yang lebih kecil.
```

Ponytail tidak boleh dipakai sebagai alasan untuk melewati validasi, test, audit log, server-side guard, emergency/red flag deterministic rule, atau disclaimer medis.
